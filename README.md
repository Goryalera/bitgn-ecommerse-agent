BitGN ECOM Agent

BitGN ECOM Agent это проект для запуска автоматического помощника в учебной и тестовой среде электронной торговли BitGN. Он получает задания от платформы, работает с отдельной виртуальной средой магазина, ищет нужные данные, выполняет проверки и отправляет итоговый ответ на оценку.

Проект создан для задач, где важно не просто дать ответ, а подтвердить его данными из среды. Агент может работать с каталогом товаров, заказами, покупателями, складскими остатками, правилами магазина, обращениями поддержки и задачами по проверке подозрительных действий.

Назначение проекта

Проект помогает автоматизировать выполнение заданий в среде BitGN ECOM. Для каждого задания создается отдельная рабочая среда, в которой находятся файлы, документы, база данных и служебные команды. Агент подключается к этой среде через защищенный набор инструментов и выполняет работу только внутри нее.

Такой подход делает выполнение заданий воспроизводимым и безопасным. Агент не использует локальные файлы пользователя для решения задач, не обращается к лишним внешним источникам и опирается на данные, которые доступны в конкретной среде задания.

Что умеет агент

Агент может читать документы и правила магазина, проверять сведения о товарах, искать данные в заказах и обращениях, работать с базой данных через встроенные команды, считать остатки и количества, проверять условия доставки и наличия, а также формировать обоснованный ответ с указанием использованных источников.

Для сложных типов задач предусмотрен выбор более сильной модели. Это используется там, где требуется точный подсчет, поиск подходящих товаров, проверка наличия или анализ подозрительных действий.

Как устроен проект

Основная точка запуска находится в файле agent/main.py. Этот файл подключается к платформе BitGN, создает запуск, получает задания, распределяет их по рабочим потокам и отправляет результаты на оценку.

Логика работы агента находится в agent/hermes_agent.py. Этот модуль готовит контекст задания, собирает инструкции, запускает модель через Hermes и передает ей доступ к среде задания.

Файл agent/ecom_mcp_server.py отвечает за инструменты, через которые модель работает с виртуальной средой магазина. Через них агент может читать файлы, искать данные, выполнять команды и сохранять ссылки на источники, которые подтверждают ответ.

Папка agent/prompts содержит основные инструкции для поведения агента. Они описывают правила работы, порядок проверки данных, выбор результата и требования к подтверждению ответа.

Папка agent/hermes_home содержит изолированную конфигурацию Hermes и дополнительные навыки для отдельных типов задач. Эти навыки помогают агенту точнее работать с проверкой чеков, анализом подозрительных операций и отчетами по каталогу.

Настройка

Для запуска нужен Python и зависимости из файла requirements.txt. Также нужны ключи доступа к BitGN и выбранному поставщику модели.

Сначала нужно создать файл .env на основе .env.example и заполнить в нем необходимые значения. В этот файл обычно добавляют ключ BitGN, ключ для модели, выбранный benchmark, название запуска и количество параллельных задач.

Файл .env не попадает в репозиторий. В нем могут храниться личные ключи, поэтому он намеренно исключен из публикации.

Запуск

После настройки окружения проект можно запускать через run.sh или напрямую из папки agent с помощью Python. При запуске агент подключается к BitGN, получает список заданий, выполняет их, отправляет результаты и ожидает итоговую оценку.

Во время работы создаются локальные журналы выполнения. Они помогают разбирать ошибки и смотреть, как агент принимал решения. Эти файлы также не отправляются в репозиторий, потому что относятся к конкретным запускам.

Что хранится в репозитории

В репозитории находятся исходный код агента, шаблоны конфигурации, инструкции, навыки, пример файла окружения, документация и вспомогательные файлы для запуска.

В репозиторий не попадают личные ключи, виртуальное окружение, временные файлы, журналы выполнения, состояние Hermes и служебные кэши. Это помогает держать проект чистым и безопасным для публикации.

Документация

Подробное описание архитектуры находится в ARCHITECTURE_OVERVIEW.md.

Описание настроек находится в CONFIGURATION.md и CONFIGURATION_GUIDE.md.

Пример переменных окружения находится в .env.example.

Итог

BitGN ECOM Agent это готовая основа для запуска агента в задачах электронной торговли BitGN. Проект объединяет подключение к платформе, работу с изолированной средой, управление моделью, сбор подтверждающих источников и отправку результатов на оценку.
